1. Prerequisites

  • Hardware Requirements:
  • CPU: 2+ cores recommended
  • RAM: 2GB minimum (8GB+ for large deployments)
  • Storage: 10GB+ for metrics
  • Network: Agent and SNMP connectivity
  • Operating System:
  • Linux: Any modern distribution (RHEL, Debian, Ubuntu, CentOS, Fedora, Arch, Alpine, openSUSE)
  • macOS: 10.14+ (Mojave or newer)
  • Windows: Windows Server 2016+ or Windows 10
  • FreeBSD: 11.0+
  • Network Requirements:
  • Port 10051 (default zabbix port)
  • Port 10050 for agents
  • Dependencies:
  • See official documentation for specific requirements
  • System Access: root or sudo privileges required

    • 2. Supported Operating Systems

    This guide supports installation on:

  • RHEL 8/9 and derivatives (CentOS Stream, Rocky Linux, AlmaLinux)
  • Debian 11/12
  • Ubuntu 20.04/22.04/24.04 LTS
  • Arch Linux (rolling release)
  • Alpine Linux 3.18+
  • openSUSE Leap 15.5+ / Tumbleweed
  • SUSE Linux Enterprise Server (SLES) 15+
  • macOS 12+ (Monterey and later)
  • FreeBSD 13+
  • Windows 10/11/Server 2019+ (where applicable)

    • 3. Installation

    RHEL/CentOS/Rocky Linux/AlmaLinux

    bash 
    # Install EPEL repository if needed
sudo dnf install -y epel-release

# Install zabbix
sudo dnf install -y zabbix-server

# Enable and start service
sudo systemctl enable --now zabbix-server

# Configure firewall
sudo firewall-cmd --permanent --add-port=10051/tcp
sudo firewall-cmd --reload

# Verify installation
zabbix_server --version

    Debian/Ubuntu

    bash 
    # Update package index
sudo apt update

# Install zabbix
sudo apt install -y zabbix-server

# Enable and start service
sudo systemctl enable --now zabbix-server

# Configure firewall
sudo ufw allow 10051

# Verify installation
zabbix_server --version

    Arch Linux

    bash 
    # Install zabbix
sudo pacman -S zabbix-server

# Enable and start service
sudo systemctl enable --now zabbix-server

# Verify installation
zabbix_server --version

    Alpine Linux

    bash 
    # Install zabbix
apk add --no-cache zabbix-server

# Enable and start service
rc-update add zabbix-server default
rc-service zabbix-server start

# Verify installation
zabbix_server --version

    openSUSE/SLES

    bash 
    # Install zabbix
sudo zypper install -y zabbix-server

# Configure firewall

    5. Service Management

    systemd (RHEL, Debian, Ubuntu, Arch, openSUSE)

    bash 
    # Enable service
sudo systemctl enable zabbix-server

# Start service
sudo systemctl start zabbix-server

sudo systemctl stop zabbix-server

# Restart service
sudo systemctl restart zabbix-server

# Check status
sudo systemctl status zabbix-server

# View logs
sudo journalctl -u zabbix-server -f

    OpenRC (Alpine Linux)

    bash 
    # Enable service
rc-update add zabbix-server default

# Start service
rc-service zabbix-server start

# Stop service
rc-service zabbix-server stop

rc-service zabbix-server restart

# Check status
rc-service zabbix-server status

service zabbix-server start

service zabbix-server stop

service zabbix-server restart

# Check status

    Reverse Proxy Setup

    nginx Configuration

    nginx 
    upstream zabbix-server_backend {
    server 127.0.0.1:10051;
}

server {
    listen 80;
    server_name zabbix-server.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name zabbix-server.example.com;

    ssl_certificate /etc/ssl/certs/zabbix-server.example.com.crt;
    ssl_certificate_key /etc/ssl/private/zabbix-server.example.com.key;

    location / {
        proxy_pass http://zabbix-server_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

    Apache Configuration

    apache 
    <VirtualHost *:80>
    ServerName zabbix-server.example.com
    Redirect permanent / https://zabbix-server.example.com/
</VirtualHost>

<VirtualHost *:443>
    ServerName zabbix-server.example.com
    
    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/zabbix-server.example.com.crt
    SSLCertificateKeyFile /etc/ssl/private/zabbix-server.example.com.key
    
    ProxyRequests Off
    ProxyPreserveHost On
    
    ProxyPass / http://127.0.0.1:10051/
    ProxyPassReverse / http://127.0.0.1:10051/
</VirtualHost>

    HAProxy Configuration

    haproxy 
    frontend zabbix-server_frontend
    bind *:80
    bind *:443 ssl crt /etc/ssl/certs/zabbix-server.pem
    redirect scheme https if !{ ssl_fc }
    default_backend zabbix-server_backend

backend zabbix-server_backend
    balance roundrobin
    server zabbix-server1 127.0.0.1:10051 check

    Basic Security Setup

sudo firewall-cmd --reload

sudo setsebool -P httpd_can_network_connect on

    9. Backup and Restore

    Backup Script

    bash 
    #!/bin/bash
# Basic backup script
BACKUP_DIR="/backup/zabbix-server"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"
tar -czf "$BACKUP_DIR/zabbix-server-backup-$DATE.tar.gz" /etc/zabbix-server /var/lib/zabbix-server

echo "Backup completed: $BACKUP_DIR/zabbix-server-backup-$DATE.tar.gz"

    Restore Procedure

    bash 
    # Stop service
sudo systemctl stop zabbix-server

# Restore from backup
tar -xzf /backup/zabbix-server/zabbix-server-backup-*.tar.gz -C /

# Start service
sudo systemctl start zabbix-server

    6. Troubleshooting

    Common Issues

    1. Service won't start:

    bash 
    # Check logs
sudo journalctl -u zabbix-server -n 100
sudo tail -f /var/log/zabbix-server/zabbix-server.log

# Check configuration
zabbix_server --version

# Check permissions
ls -la /etc/zabbix-server

    2. Connection issues:

    bash 
    # Check if service is listening
sudo ss -tlnp | grep 10051

# Test connectivity
telnet localhost 10051

# Check firewall
sudo firewall-cmd --list-all

    3. Performance issues:

    bash 
    # Check resource usage
top -p $(pgrep zabbix-server)

# Check disk I/O
iotop -p $(pgrep zabbix-server)

# Check connections
ss -an | grep 10051

    Integration Examples

    Docker Compose Example

    yaml 
    version: '3.8'
services:
  zabbix-server:
    image: zabbix-server:latest
    ports:
      - "10051:10051"
    volumes:
      - ./config:/etc/zabbix-server
      - ./data:/var/lib/zabbix-server
    restart: unless-stopped

    Maintenance

    Update Procedures

    bash 
    # RHEL/CentOS/Rocky/AlmaLinux
sudo dnf update zabbix-server

# Debian/Ubuntu
sudo apt update && sudo apt upgrade zabbix-server

# Arch Linux
sudo pacman -Syu zabbix-server

# Alpine Linux
apk update && apk upgrade zabbix-server

# openSUSE
sudo zypper update zabbix-server

# FreeBSD
pkg update && pkg upgrade zabbix-server

# Always backup before updates
tar -czf /backup/zabbix-server-pre-update-$(date +%Y%m%d).tar.gz /etc/zabbix-server

# Restart after updates
sudo systemctl restart zabbix-server

    Regular Maintenance

    bash 
    # Log rotation
sudo logrotate -f /etc/logrotate.d/zabbix-server

# Clean old logs
find /var/log/zabbix-server -name "*.log" -mtime +30 -delete

# Check disk usage
du -sh /var/lib/zabbix-server
