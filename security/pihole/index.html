Pi-hole

pihole is a free and open-source network-wide ad blocker. Pi-hole acts as a DNS sinkhole to block ads and trackers network-wide, providing privacy and performance benefits

Pi-hole Installation Guide

Network-wide ad and tracker blocking DNS sinkhole that protects your entire network from unwanted content without requiring client-side software. Essential tool for privacy, security, and performance improvement.

Prerequisites

  • Linux system (any modern distribution)
  • Root or sudo access
  • 1GB RAM minimum, 2GB recommended
  • Static IP address for DNS server
  • Port 53 available for DNS service
  • Web server port (80/443) available for admin interface

    • Installation

    One-Line Installation (Recommended)

    bash 
    # Download and run the automated installer
curl -sSL https://install.pi-hole.net | bash

# Alternative: Download and review before running
wget -O basic-install.sh https://install.pi-hole.net
less basic-install.sh  # Review the script
sudo bash basic-install.sh

    Ubuntu/Debian Manual Installation

    bash 
    # Update system packages
sudo apt update && sudo apt upgrade -y

# Install dependencies
sudo apt install -y curl dnsutils net-tools

# Create pihole user
sudo useradd --system --shell /usr/sbin/nologin pihole

# Download Pi-hole
git clone --depth 1 https://github.com/pi-hole/pi-hole.git Pi-hole
cd Pi-hole

# Run installer
sudo bash automated\ install/basic-install.sh

# Verify installation
pihole status
dig @localhost google.com

    Configuration

    Enhanced Privacy Setup

    bash 
    # Configure maximum privacy
sudo tee /etc/pihole/pihole-FTL.conf > /dev/null <<EOF
PRIVACYLEVEL=2
MAXLOGAGE=24
IGNORE_LOCALHOST=yes
BLOCKINGMODE=IP-AAAA-NODATA
CNAME_INSPECTION=yes
EOF

sudo systemctl restart pihole-FTL

    Custom Blocklists

    bash 
    # Add comprehensive blocklists
pihole -a -l https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts
pihole -a -l https://mirror1.malwaredomains.com/files/justdomains
pihole -a -l https://s3.amazonaws.com/lists.disconnect.me/simple_tracking.txt

# Update blocklists
pihole -g

    Security and Monitoring

    Security Hardening

    bash 
    # Secure web interface
sudo tee /etc/lighttpd/conf-available/99-pihole-security.conf > /dev/null <<EOF
server.modules += ( "mod_setenv" )
setenv.add-response-header = (
    "X-Frame-Options" => "DENY",
    "X-Content-Type-Options" => "nosniff",
    "Strict-Transport-Security" => "max-age=31536000"
)
EOF

sudo lighttpd-enable-mod pihole-security
sudo systemctl restart lighttpd

    Monitoring Script

    bash 
    sudo tee /usr/local/bin/pihole-monitor.sh > /dev/null <<'EOF'
#!/bin/bash
LOG="/var/log/pihole-monitor.log"

log_message() {
    echo "$(date '+%Y-%m-%d %H:%M:%S') - $1" | tee -a ${LOG}
}

# Check service
if systemctl is-active pihole-FTL >/dev/null; then
    log_message "✓ Pi-hole service running"
else
    log_message "✗ Pi-hole service not running"
fi

# Check DNS resolution
if dig @localhost google.com +short >/dev/null 2>&1; then
    log_message "✓ DNS resolution working"
else
    log_message "✗ DNS resolution failed"
fi

# Check blocking
if dig @localhost doubleclick.net +short | grep -q "0.0.0.0"; then
    log_message "✓ Ad blocking working"
else
    log_message "⚠ Ad blocking issues"
fi

log_message "Pi-hole monitoring completed"
EOF

sudo chmod +x /usr/local/bin/pihole-monitor.sh
echo "*/15 * * * * root /usr/local/bin/pihole-monitor.sh" | sudo tee -a /etc/crontab

    Backup Strategy

    bash 
    sudo tee /usr/local/bin/pihole-backup.sh > /dev/null <<'EOF'
#!/bin/bash
BACKUP_DIR="/backup/pihole"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p ${BACKUP_DIR}

# Backup Pi-hole configuration
pihole -a -t > ${BACKUP_DIR}/pihole-backup-${DATE}.tar.gz

# Keep last 14 backups
find ${BACKUP_DIR} -name "pihole-backup-*.tar.gz" -mtime +14 -delete

echo "Pi-hole backup completed: ${DATE}"
EOF

sudo chmod +x /usr/local/bin/pihole-backup.sh
echo "0 2 * * * root /usr/local/bin/pihole-backup.sh" | sudo tee -a /etc/crontab
